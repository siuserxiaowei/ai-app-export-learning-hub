# 7 天学习计划：从理解 AI App 出海到选出一个方向

适合：个人自学、社群打卡、训练营预习  
节奏：每天 30-60 分钟  
产出：一个可验证的 AI App 出海方向

## Day 1：建立正确认知

阅读：`docs/learning-guide.md` 的前半部分。  
目标：理解出海不是翻译，AI 功能不等于产品。

作业：

```text
我为什么想做出海？
我是因为海外有明确需求，还是因为国内太卷？
```

## Day 2：选择具体用户

阅读：判断方向的 7 个问题。  
目标：把宽泛人群收窄成具体人群。

作业：

```text
我的目标用户是：
他们每周会遇到的具体任务是：
他们现在怎么解决：
```

## Day 3：拆一个竞品

阅读：`docs/case-study-workbook.md` 中任意 2 个案例。  
目标：学习如何从竞品中提取可迁移规律。

作业：

```text
竞品名称：
它服务谁：
它替代了什么旧流程：
我能迁移到自己的方向的一点是：
```

## Day 4：定义产品一句话

阅读：公开内容包里的“三个推荐方向”。  
目标：把想法写成一句清楚的产品定义。

模板：

```text
我帮助 [具体用户] 在 [具体场景] 中，用 AI 更快完成 [具体结果]。
```

## Day 5：做 MVP 边界

目标：只定义第一版最小可交付结果。

作业：

```text
用户输入：
AI 处理：
最终输出：
第一个付费点：
不要做的功能：
```

## Day 6：找第一个渠道

目标：找一个不用大额买量也能触达目标用户的渠道。

可选渠道：

- SEO 长尾文章
- TikTok/Reels 结果展示
- Reddit/社群帖子
- YouTube 教程
- 冷邮件
- 创作者合作

作业：

```text
第一个渠道：
第一条内容/消息：
目标转化动作：
```

## Day 7：做继续/停止判断

目标：判断是否值得进入 MVP 开发。

继续条件：

- 至少 5 个目标用户愿意聊
- 至少 1-3 个用户表达明确付费意向
- 你能说清楚第一个渠道和第一个付费点

停止或调整条件：

- 用户太宽泛
- 痛点不强
- 只能靠买量
- 结果不可展示
- 合规风险明显高于收益
