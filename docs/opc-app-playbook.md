# OPC AI App 出海方向剧本

适合人群：OPC、超级个体、独立开发者、小团队创始人

学习目标：把“AI App 出海可以做什么”变成 10 个可验证方向，每个方向都有目标用户、旧流程、MVP、首个渠道、7 天验证动作和淘汰标准。

使用方式：先读“筛选原则”，再选 2 个方向做对比，最后把自己的方向按同一模板重写。

## 筛选原则

OPC 不应该先问“我能做什么功能”，而应该问：

```text
谁现在有一个重复、麻烦、可付费的任务？
AI 能不能把这个任务压缩到 1 个输入、1 个流程、1 个可交付结果？
我能不能不用大规模买量，先找到 10 个真实用户？
```

进入验证的方向至少要满足 4 条：

1. 用户足够具体。
2. 旧流程已经存在，且用户愿意花时间或钱解决。
3. MVP 可以在 7-14 天内做成可演示版本。
4. 第一个渠道不依赖大规模广告。
5. 输出结果能截图、导出、分享或直接用于工作。
6. 合规边界可控。
7. 模型成本不会吃掉毛利。

## 方向 1：Etsy / TikTok Shop Listing Video Kit

一句话：帮助 Etsy、TikTok Shop、Shopify 小卖家，把 3-5 张商品图变成英文 UGC 广告脚本、短视频结构、caption 和 listing 标题变体。

### 为什么值得看

跨境卖家有明确 ROI：素材影响上架速度、广告效果和转化。这个方向不卖“AI 视频生成”，而卖“我今天能多出几条可测试的商品广告”。

### 目标用户

- Etsy 饰品、手作、家居小卖家。
- TikTok Shop 小商家。
- Shopify DTC 小团队。
- 没有专职设计和视频团队的人。

### 旧流程

```text
拍商品图 → 想标题 → 写广告脚本 → 做短视频结构 → 写 caption → 反复改
```

### MVP

输入：

- 商品图。
- 商品标题。
- 目标平台。
- 卖点关键词。

输出：

- 3 个 hook。
- 3 条 UGC 广告脚本。
- 2 个镜头结构。
- 5 个 caption。
- 5 个 listing title variants。

### 首个渠道实验

1. 做 5 个真实商品 before/after 案例。
2. 发到 TikTok / Reels / 小卖家社区。
3. 做一个 “free Etsy ad script generator” 长尾 SEO 页面。
4. 冷私信 30 个 Etsy/TikTok Shop 小卖家。

### 7 天验证

| 天数 | 动作 |
| --- | --- |
| Day 1 | 找 20 个卖家真实商品页，手动生成素材。 |
| Day 2 | 做 5 个前后对比样例。 |
| Day 3 | 做 landing page 和 demo 表单。 |
| Day 4 | 发 3 条短视频演示。 |
| Day 5 | 私信 30 个卖家，送一次免费生成。 |
| Day 6 | 让 5 个卖家上传商品图。 |
| Day 7 | 测试 3 个卖家是否愿意为导出包付费。 |

### 淘汰标准

- 卖家不愿意提供真实商品图。
- 生成结果不能直接改成可发布素材。
- 用户只觉得好玩，不愿为导出、批量或模板付费。

## 方向 2：InterviewLoop

一句话：帮助非英语母语求职者，做岗位化模拟面试、答案评分、表达改写和复习计划。

### 为什么值得看

留学生、转码求职者、国际岗位申请人有强动机，且能接受持续练习。产品不应该做泛英语陪练，而应围绕“拿 offer”这个高价值结果。

### 目标用户

- 非英语母语软件工程师候选人。
- 国际学生。
- Career switcher。
- 准备英语行为面试的人。

### 旧流程

```text
找题库 → 自己练 → 不知道答案好不好 → 找人 mock → 记录反馈 → 复习
```

### MVP

输入：

- 目标岗位。
- 面试类型。
- 用户回答文字或语音。

输出：

- 答案结构评分。
- 英文表达改写。
- STAR 结构反馈。
- 追问模拟。
- 7 天复习计划。

### 首个渠道实验

1. 做 20 个岗位题 SEO 页面。
2. 在 Reddit career / csMajors / cscareerquestions 相关主题下做非硬广回复。
3. 做 YouTube Shorts / TikTok “bad answer vs better answer”。
4. 找留学生社群测试。

### 7 天验证

| 天数 | 动作 |
| --- | --- |
| Day 1 | 选一个岗位，例如 frontend engineer。 |
| Day 2 | 整理 30 个高频题。 |
| Day 3 | 手动给 5 个答案做反馈样例。 |
| Day 4 | 做一个免费评分页面。 |
| Day 5 | 找 20 个候选人测试。 |
| Day 6 | 看是否有人连续练 3 次。 |
| Day 7 | 测试付费：完整 mock + 复习计划。 |

### 淘汰标准

- 用户不愿上传真实回答。
- 反馈太泛，和 ChatGPT 差异不明显。
- 第 7 天没有用户回来继续练习。

## 方向 3：ClientBriefAI

一句话：帮助自由职业者和小 agency，把客户零散需求整理成 proposal、scope、timeline、报价档位和跟进邮件。

### 为什么值得看

自由职业者愿意为节省提案时间和提高成交率付费。这个方向更适合 web-first，不必先做移动 App。

### 目标用户

- 设计师。
- Web freelancer。
- 小型 marketing agency。
- 独立顾问。

### 旧流程

```text
客户发一堆消息 → 手动整理需求 → 写方案 → 估工期 → 做报价 → 写 follow-up
```

### MVP

输入：

- 客户聊天记录。
- 项目背景。
- 预算范围。
- 服务类型。

输出：

- Proposal。
- Scope table。
- Timeline。
- 3 个报价档位。
- Risk notes。
- Follow-up email。

### 首个渠道实验

1. 做 5 个 proposal 模板页面。
2. LinkedIn 发 freelancer proposal teardown。
3. Product Hunt 做免费小工具。
4. 冷邮件联系 50 个 freelancer。

### 7 天验证

| 天数 | 动作 |
| --- | --- |
| Day 1 | 找 5 个真实 freelancer 访谈。 |
| Day 2 | 收集 10 份匿名 brief。 |
| Day 3 | 手动生成 proposal 样例。 |
| Day 4 | 做一个 paste-to-proposal demo。 |
| Day 5 | 让 10 个用户导出 PDF。 |
| Day 6 | 测试品牌模板付费。 |
| Day 7 | 看 2 个用户是否愿意用于真实客户。 |

### 淘汰标准

- 用户不愿粘贴真实需求。
- 输出不能明显减少提案时间。
- 用户只需要一次性模板，不需要持续使用。

## 方向 4：CourseClipper

一句话：帮助课程创作者、播客和教练，把长视频/音频变成短视频选题、摘要、newsletter 和社媒 caption。

### MVP

输入：视频/音频链接、转写文本或大纲。  
输出：5 个高光片段建议、短视频标题、caption、newsletter 摘要、发布时间建议。

### 首个渠道

- YouTube creator SEO。
- 创作者社群。
- Affiliate creator。
- “长视频变 10 条短内容”前后对比演示。

### 淘汰标准

- 用户不发布生成内容。
- 只想要免费摘要，不愿为批量工作流付费。
- 与 OpusClip、Descript、Captions 差异不明显。

## 方向 5：ContractBrief for Creators

一句话：帮助创作者和小 agency 看懂合作合同里的付款、交付、版权、使用权和违约风险。

### MVP

输入：合同 PDF 或条款文本。  
输出：plain-language brief、风险点、谈判邮件、交付清单。

### 注意边界

这个方向不能提供法律意见。产品定位应是“合同理解辅助”和“沟通准备”，不是律师替代。

### 首个渠道

- Creator community。
- “sponsorship contract checklist” SEO。
- Agency 冷启动访谈。

### 淘汰标准

- 合规免责声明无法讲清。
- 用户期待法律结论而不是辅助理解。
- 需要强律师背书才能转化。

## 方向 6：Amazon Listing QA

一句话：帮助 Amazon / Shopify 卖家检查标题、bullet points、图片、claim、关键词覆盖和上架前风险。

### MVP

输入：listing 文案、图片、目标品类。  
输出：SEO checklist、claim risk、rewrite suggestions、image checklist、launch readiness score。

### 首个渠道

- Amazon listing checklist SEO。
- Seller forum。
- 冷邮件给小卖家。
- 免费 listing review。

### 淘汰标准

- 用户只信 Helium 10 / Jungle Scout 等成熟工具。
- 产品变成完整 seller analytics suite，超出 OPC 范围。
- 平台规则变动导致维护成本过高。

## 方向 7：Sleep Journal Coach

一句话：帮助高压力知识工作者做睡前记录、压力复盘和轻量睡眠改善计划。

### MVP

输入：睡前文字/语音日记、睡眠感受。  
输出：情绪总结、睡前建议、每周 pattern report、轻量提醒。

### 注意边界

不要做医疗睡眠诊断，不承诺治疗失眠。定位应是 journaling、habit 和 reflection。

### 首个渠道

- App Store Search。
- TikTok wellness demos。
- Newsletter creator affiliate。

### 淘汰标准

- 第 7 天留存低。
- 用户不愿长期记录。
- 需要医疗可信度才能转化。

## 方向 8：DevDocs Fix Assistant

一句话：帮助独立开发者和小 SaaS 团队，把错误日志、文档和上下文转成可执行修复步骤和 PR-ready notes。

### MVP

输入：错误日志、相关代码片段、框架版本。  
输出：可能原因、修复步骤、文档链接、patch checklist、PR 描述。

### 首个渠道

- GitHub issue 示例。
- Hacker News / Reddit 复盘。
- Dev SEO。
- 开源项目免费小工具。

### 淘汰标准

- 和 ChatGPT / Cursor 差异不明显。
- 缺少开发者信任。
- 需要完整 IDE 才能成立。

## 方向 9：Creator Voice Localizer

一句话：帮助 YouTuber、课程创作者和独立教育者，把脚本转成多语言字幕、配音脚本和 voiceover draft。

### MVP

输入：脚本或音频。  
输出：localized script、subtitle file、voiceover draft、pronunciation notes。

### 注意边界

不要自研底层语音模型。优先接现有 TTS / voice API，重点做行业模板、工作流和发布包。

### 首个渠道

- YouTube SEO。
- Creator affiliate。
- 小语种创作者社区。

### 淘汰标准

- 用户担心声音授权或版权。
- 输出质量不够发布。
- 成本结构无法支持低价订阅。

## 方向 10：Fitness Meal Photo Logger

一句话：帮助健身和减脂新手，用照片记录饮食并获得轻量估算和习惯反馈。

### MVP

输入：餐食照片、目标、用户修正。  
输出：热量/宏量估算、每日总结、习惯 streak、轻量提醒。

### 注意边界

不要承诺临床准确性，不提供医疗建议。需要明确这是估算和习惯辅助。

### 首个渠道

- TikTok/Reels meal logging demos。
- App Store Search。
- Fitness creator affiliate。

### 淘汰标准

- 用户对准确性投诉高。
- 订阅退款多。
- 获客只能靠高成本买量。

## 推荐优先级

如果你是一个人或两三人小团队，我建议优先级是：

| 优先级 | 方向 | 原因 |
| --- | --- | --- |
| 1 | Listing Video Kit | 商业结果明确，输出可展示，用户懂 ROI。 |
| 2 | ClientBriefAI | Web-first，合规低，容易手动验证。 |
| 3 | InterviewLoop | 需求强，适合内容/SEO，但留存要验证。 |
| 4 | Amazon Listing QA | 小 B 付费清楚，但平台规则维护要控制。 |
| 5 | CourseClipper | 需求强但红海，必须切人群。 |

谨慎方向：

- Fitness Meal Photo Logger：需求大，但健康、准确性、退款和买量风险高。
- ContractBrief：价值高，但法律边界要非常清楚。
- Creator Voice Localizer：需求真实，但声音授权、成本和质量门槛高。

```text
Free sample

```

## 最后提醒

OPC 做 AI App 出海，最危险的不是不会开发，而是：

1. 用户太泛。
2. 任务太泛。
3. 输出不可直接使用。
4. 没有第一个低成本渠道。
5. 没有明确淘汰标准。

每一个方向都应该先变成 7 天验证，而不是先变成 3 个月开发计划。
