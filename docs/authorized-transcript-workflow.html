授权逐字稿处理流程

用途：当用户提供自己拥有版权、已获得授权、或明确可处理的完整视频文本时，用本流程生成私有逐字稿、翻译稿、学习注释和公开版摘要。

适用场景：

  • 用户自己录制的视频。
  • 用户已经获得原作者授权的视频文本。
  • 用户手动提供并确认可用于处理的完整文本。
  • 明确属于可自由使用范围的文本。

不适用场景：

  • 只给一个外部平台视频链接。
  • 未授权要求复现完整字幕、完整逐字稿或完整逐句翻译。
  • 要把外部视频的完整表达公开发布到 GitHub Pages。

目录约定

公开仓库只保存合规学习材料。完整授权文本放在本地私有目录，不推送到 GitHub。

private/
  transcript-source-authorized.md
  transcript-translation-authorized.md
  transcript-notes-authorized.md

private/ 已加入 .gitignore，用于避免把完整授权文本误推到公开仓库。

用户授权确认模板

处理完整逐字稿前，先让用户明确下面 4 件事：

1. 这份文本/视频我有权提供给你处理。
2. 我允许你在本地工作区处理完整文本。
3. 我不要求把完整逐字稿或完整逐句译文发布到公开 GitHub Pages，除非我能证明拥有公开发布权。
4. 公开资料包只保留摘要、短摘录、学习稿、术语表、案例分析和练习。

输入格式

推荐用户提供如下格式：

标题：
来源：
授权说明：
是否允许公开完整文本：是/否

[00:00-00:30]
原文内容...

[00:30-01:00]
原文内容...

如果用户没有时间戳，也可以提供纯文本。处理时先按语义段落切分，再补充时间段或章节编号。

处理步骤

第 1 步：建立私有源文件

将授权文本放入：

private/transcript-source-authorized.md

文件头部保留：

授权来源：
授权范围：
是否可公开：
处理日期：
处理人：

第 2 步：做逐字稿校对

检查：

  • 人名、品牌名、产品名。
  • 数字、金额、年份、时间。
  • 英文缩写，例如 OPC、LTV、CAC、ROI、GDPR。
  • 明显误听词。
  • 口头语是否保留。

校对后的文件放入：

private/transcript-source-authorized.md

第 3 步：做逐句或分段翻译

如果授权允许完整翻译，生成：

private/transcript-translation-authorized.md

推荐结构：

## 00:00-00:30

原文：
...

译文：
...

术语：
- ...

第 4 步：做学习注释

把文本转成学习材料：

private/transcript-notes-authorized.md

每段包含：

项目内容
原观点这一段说了什么
产品启发对 OPC / 小团队有什么用
风险边界哪些不能照搬
可迁移问题学习者要回答什么
英文表达可用于出海表达的英文句式

第 5 步：生成公开版材料

即使有私有完整文本，公开仓库仍默认只发布：

  • 时间轴摘要。
  • 分段转述。
  • 英文摘要。
  • 英文学习稿。
  • 术语表。
  • 短摘录。
  • 案例和品牌拆解。
  • 作业模板。

公开版更新这些文件：

docs/source-brief.md
docs/bilingual-study-note.md
docs/learning-guide.md
docs/public-content-pack.md
docs/final-completion-audit.md

验收清单

完整授权处理完成后，检查：

检查项通过标准
授权说明private/ 源文件头部有授权范围。
私有保护完整逐字稿和完整译文不在公开仓库。
术语准确品牌名、产品名、英文缩写已校对。
翻译一致同一术语前后一致。
学习价值每段都有产品启发和迁移问题。
公开边界GitHub Pages 不发布未授权完整文本。
验证脚本python3 scripts/verify_package.py 通过。

发布判断

内容默认是否公开条件
完整逐字稿否只有明确公开授权时才可公开。
完整逐句译文否只有明确公开授权时才可公开。
时间轴摘要是不复现连续表达。
分段转述是使用自己的教学化表达。
英文学习稿是不逐句对照原文。
术语表是原创整理。
案例和品牌拆解是基于公开资料和原创分析。

下一步

如果用户提供授权文本，下一轮可以直接执行：

1. 将文本放入 private/transcript-source-authorized.md。
2. 校对专有名词和时间段。
3. 生成 private/transcript-translation-authorized.md。
4. 生成 private/transcript-notes-authorized.md。
5. 更新公开版摘要和学习资料。
6. 运行验证并提交公开资料包更新。
