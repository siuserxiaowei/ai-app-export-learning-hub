# AI App 出海竞品案例学习册

用途：给 OPC、超级个体、独立开发者学习竞品，不是看热闹  
使用方式：每个案例按 10-15 分钟学习，先读案例，再做讨论题和迁移练习  
核心问题：这个品牌为什么能成立？它对小团队有什么可学、有什么不能学？

## 学习方法

看 AI App 出海竞品，不要只看表面功能。

很多人拆竞品只会看：

- 它有什么功能
- UI 好不好看
- 价格是多少
- 用了什么模型

这不够。

真正要看的是：

```text
它服务谁？
它替代了什么旧流程？
它把哪一步变快了？
它为什么能收费？
它靠什么获客？
它有什么信任门槛？
它有哪些地方不适合小团队模仿？
```

下面每个案例都按这个结构来拆。

## 案例 1：Cal AI

### 它是什么

Cal AI 是一个用拍照估算食物热量和营养的 App。

用户不用手动搜索食物、输入克数、逐项记录，只需要拍照，系统就给出热量和营养估算，再让用户确认或修正。

### 它服务谁

主要服务：

- 减脂用户
- 健身用户
- 想控制饮食但讨厌手动记录的人
- 年轻移动端用户

### 它替代了什么旧流程

传统饮食记录流程是：

```text
打开 App → 搜索食物 → 选择品牌/种类 → 输入重量 → 记录 → 复盘
```

Cal AI 把流程压缩为：

```text
拍照 → 估算 → 修正 → 记录
```

### 它为什么能吸引用户

因为它抓住了一个非常强的痛点：

```text
用户不是不知道记录饮食有用，而是太麻烦，坚持不下去。
```

AI 在这里的价值不是“更聪明”，而是“更省事”。

### 小团队可以学什么

1. 找高频但麻烦的动作。
2. 用 AI 减少用户输入。
3. 做一个结果非常直观的体验。
4. 用短视频展示“拍一下就完成”的爽感。

### 小团队不要学什么

不要以为这个方向只是“接一个识图模型”。

难点在：

- 准确性
- 健康合规
- 用户信任
- 订阅退款
- 买量成本
- 长期留存

如果没有增长能力，直接复制 Cal AI 很可能烧钱。

### 迁移练习

请回答：

```text
你的目标用户有没有一个“知道该做但懒得做”的高频动作？
能不能把它压缩成一次拍照、一次上传、一次粘贴或一次语音？
```

## 案例 2：Photoroom

### 它是什么

Photoroom 是面向商品图和电商视觉素材的 AI 图片工具。

它不是泛修图工具，而是围绕卖货场景提供背景移除、商品图生成、批量处理、阴影、模板和 API。

### 它服务谁

主要服务：

- 电商卖家
- 小品牌
- marketplace 商家
- 营销团队
- 需要大量商品图的人

### 它替代了什么旧流程

旧流程是：

```text
拍照 → 找设计师修图 → 换背景 → 做多平台尺寸 → 上架
```

Photoroom 把流程变成：

```text
上传商品图 → 自动抠图/生成背景 → 批量导出 → 上架
```

### 它为什么能收费

因为商品图直接影响销售。

用户付费不是为了“图片好看”，而是为了：

- 更快上架
- 降低摄影成本
- 提高广告和 listing 质量
- 批量处理商品

### 小团队可以学什么

1. 不要做泛图片工具，要绑定商业场景。
2. 用户愿意为“能卖货”的素材付费。
3. 批量处理是小 B 工具的重要价值。
4. API 和工作流集成可以提高客单价。

### 小团队不要学什么

不要一上来做完整设计平台。

更适合小团队的切法是：

```text
只做某个平台、某类商品、某种素材。
```

例如：

- Etsy 饰品图
- Shopify 服装图
- TikTok Shop 商品短视频封面
- Amazon 主图合规检查

### 迁移练习

请写出：

```text
我想服务的卖家类型：
他们最常需要的 3 种素材：
我能先自动化的 1 个素材：
```

## 案例 3：Gamma

### 它是什么

Gamma 是一个用 AI 生成 presentation、document、webpage 的表达型工具。

它不是简单的 PPT 生成器，而是帮助用户把想法变成结构化、可展示的表达物。

### 它服务谁

主要服务：

- 创业者
- 市场团队
- 教育工作者
- 咨询顾问
- 知识工作者

### 它替代了什么旧流程

旧流程是：

```text
想大纲 → 写内容 → 找模板 → 排版 → 调样式 → 导出
```

Gamma 把流程变成：

```text
输入主题 → 生成结构 → 生成页面 → 在线编辑 → 分享
```

### 它为什么能成立

因为很多人不是不会写字，而是不会组织表达。

Gamma 的价值在于：

- 给结构
- 给视觉
- 给可分享页面
- 降低从想法到表达物的门槛

### 小团队可以学什么

1. 用户买的是表达成果，不是幻灯片文件。
2. 结构化输出比单段文本更有价值。
3. 分享链接本身可以带来传播。
4. 模板和场景会降低用户决策成本。

### 小团队不要学什么

不要做泛 PPT 生成器。

大平台一定会做：

- Google Slides AI
- Microsoft PowerPoint Copilot
- Canva AI
- Notion AI

小团队应该切垂直场景：

- 咨询顾问 proposal
- 创业者 investor update
- 老师课程讲义
- YouTuber 课程 slide

### 迁移练习

请回答：

```text
你的用户需要把什么想法表达给谁？
最终交付物是 deck、网页、PDF、邮件，还是 proposal？
```

## 案例 4：PDF.ai / ChatPDF 类

### 它是什么

这类产品让用户上传 PDF，然后进行问答、总结、提取信息。

### 它服务谁

泛用户包括：

- 学生
- 研究人员
- 法律/金融/保险从业者
- 企业知识工作者

### 它替代了什么旧流程

旧流程是：

```text
打开 PDF → 搜索关键词 → 读长文 → 手动整理 → 写摘要
```

AI PDF 工具把流程变成：

```text
上传 PDF → 提问 → 得到摘要/答案
```

### 它为什么早期容易增长

因为用户非常容易理解：

```text
上传文件，然后问它问题。
```

教育成本低，传播也简单。

### 小团队可以学什么

文档是非常好的 AI 入口，因为：

- 用户已有材料
- 任务明确
- 输出可以验证
- 很多行业文件很复杂

### 小团队不要学什么

不要再做泛 PDF Chat。

因为这个能力会被：

- ChatGPT
- Gemini
- Claude
- Adobe Acrobat
- 浏览器
- 办公套件

不断内置。

更好的切法是行业文件工作流：

- 合同风险解释
- 保险理赔材料检查
- 移民材料清单
- 课程讲义转测验
- 采购合同对比

### 迁移练习

请写：

```text
我的用户最常处理哪类文件？
他读文件的目的是什么？
AI 输出应该是摘要、风险清单、决策建议，还是回复邮件？
```

## 案例 5：HeyGen / Synthesia

### 它是什么

HeyGen 和 Synthesia 属于 AI 视频、数字人、视频本地化方向。

它们帮助用户生成多语言视频、虚拟讲解、营销内容和培训材料。

### 它服务谁

主要服务：

- 企业营销团队
- 培训团队
- 销售团队
- 跨国公司
- 内容创作者

### 它替代了什么旧流程

旧流程是：

```text
写脚本 → 找演员 → 拍摄 → 剪辑 → 翻译 → 配音 → 多语言版本
```

AI 视频工具把部分流程压缩为：

```text
输入脚本 → 选择 avatar/声音 → 生成视频 → 本地化
```

### 它为什么有价值

因为视频生产贵、慢、协作复杂。

尤其是多语言视频，传统方式成本很高。

### 小团队可以学什么

1. 视频本地化是真需求。
2. 企业用户为节省制作成本付费。
3. 多语言、多版本、规模化生产是高价值场景。

### 小团队不要学什么

不要做底层数字人平台。

这类产品涉及：

- 模型能力
- 肖像授权
- deepfake 合规
- 企业安全
- 渲染成本
- 品牌信任

小团队更适合做服务层：

```text
用现有 API 组合出某行业的视频本地化工作流。
```

例如：

- 课程视频翻译和字幕包
- YouTube 视频本地化
- 产品教程多语言版本
- 销售视频脚本和配音包

### 迁移练习

请回答：

```text
我的目标用户有没有“同一内容生成多语言、多平台、多版本”的需求？
我能不能先不生成视频，只生成脚本、字幕和配音稿？
```

## 案例 6：Lovart

### 它是什么

Lovart 是一个 AI design agent，强调从一个需求生成多种视觉资产。

它不是单张图片生成，而是围绕品牌、海报、视频、视觉方案等做完整设计工作流。

### 它服务谁

主要服务：

- 设计师
- 品牌团队
- 创作者
- 市场团队
- 需要快速出视觉方案的人

### 它替代了什么旧流程

旧流程是：

```text
收集需求 → 做 moodboard → 做 logo/主视觉 → 做海报 → 做社媒图 → 做视频素材
```

Lovart 试图把流程变成：

```text
描述需求 → 生成视觉方向 → 生成多资产 → 继续编辑
```

### 它为什么值得学

因为它体现了 Agent 产品的一个关键点：

```text
Agent 不是聊天框，而是替用户完成一串专业决策。
```

### 小团队可以学什么

1. 多资产联动比单张图更有价值。
2. 用户真正要的是视觉方案，不是 prompt。
3. Agent 要能处理上下文和连续决策。

### 小团队不要学什么

不要一开始做全能设计 Agent。

小团队可以收窄为：

- Shopify 店铺首屏视觉包
- 播客封面和社媒宣传图
- 独立咨询师品牌物料
- 课程发布素材包

### 迁移练习

请写：

```text
我的用户一次发布/销售/展示，需要哪些配套素材？
我能不能一次生成 3-5 个相关资产，而不是只生成一个？
```

## 案例 7：Captions / CapCut

### 它是什么

Captions 和 CapCut 都围绕短视频创作，提供字幕、剪辑、翻译、AI actor、音频、模板、平台适配等能力。

### 它服务谁

主要服务：

- 短视频创作者
- 小商家
- 内容团队
- 直播和电商团队
- 个人品牌创作者

### 它替代了什么旧流程

旧流程是：

```text
拍摄 → 导入剪辑软件 → 加字幕 → 加 B-roll → 调声音 → 改尺寸 → 发布
```

AI 视频工具把流程压缩为：

```text
上传素材 → 自动字幕/剪辑/翻译 → 导出多平台版本
```

### 它为什么强

因为短视频生产是高频需求。

创作者每天都要面对：

- 没选题
- 没脚本
- 剪辑慢
- 字幕麻烦
- 多平台尺寸不同
- 标题封面要优化

### 小团队可以学什么

短视频方向要做“平台 + 人群 + 任务”。

例如：

```text
TikTok Shop 卖家的商品短视频广告
YouTube 教育博主的课程切片
播客主的社媒预告片
```

### 小团队不要学什么

不要做泛剪辑器。

泛剪辑器会遇到：

- 巨头竞争
- 功能复杂
- 用户预期高
- 模型和渲染成本高

### 迁移练习

请回答：

```text
我的视频用户发布在哪个平台？
他最痛的是脚本、剪辑、字幕、翻译、封面，还是分发？
```

## 案例 8：Photo AI / Interior AI

### 它是什么

Photo AI 和 Interior AI 是 Pieter Levels 代表性的独立开发者 AI 产品。

它们分别围绕 AI 人像照片和室内设计效果生成。

### 它服务谁

Photo AI 服务：

- 需要头像照的人
- 创作者
- 远程工作者
- 个人品牌用户

Interior AI 服务：

- 房东
- 室内设计爱好者
- 房产展示用户
- 想快速看装修效果的人

### 它为什么值得学

因为它们体现了 OPC 的一个重要优势：

```text
一个人可以用强产品感、强分发和快速迭代，做出可赚钱的小产品。
```

### 小团队可以学什么

1. 可展示结果很重要。
2. 创始人个人品牌是分发资产。
3. SEO 和社交传播可以替代部分买量。
4. 产品可以先很窄，再慢慢优化。

### 小团队不要学什么

不要只学“一个人做产品”。

Pieter Levels 的优势不只是会写代码，还包括：

- 长期公开构建
- 大量受众
- 快速发布能力
- 会讲故事
- 会做 SEO

如果没有这些分发资产，不能照搬结果。

### 迁移练习

请写：

```text
我的产品结果是否适合前后对比？
我有没有一个可以长期公开构建的内容主题？
```

## 案例 9：Calm / Wysa

### 它是什么

Calm 是睡眠、冥想和放松内容产品。  
Wysa 是偏 AI 心理支持和职场心理健康的产品。

### 它服务谁

主要服务：

- 压力大的人
- 睡眠不好的人
- 职场情绪困扰用户
- 企业员工健康项目

### 它为什么能收费

这类产品卖的不是 AI，而是：

- 安全感
- 陪伴
- 稳定习惯
- 情绪缓解
- 内容质量

### 小团队可以学什么

1. 情绪类产品要建立信任。
2. 内容和体验比技术炫技更重要。
3. 长期留存来自习惯，而不是一次惊艳生成。
4. 企业渠道可能比纯 C 端订阅更稳定。

### 小团队不要学什么

不要轻易承诺治疗。

心理健康方向风险包括：

- 医疗边界
- 危机干预
- 隐私数据
- 错误建议
- 用户依赖

小团队更适合从低风险场景切入：

- journaling
- 睡前复盘
- 职场压力记录
- 情绪模式总结

### 迁移练习

请回答：

```text
我的产品是在提供治疗，还是提供记录、反思、陪伴和教育？
用户最敏感的数据是什么，我如何保护它？
```

## 案例 10：ElevenLabs

### 它是什么

ElevenLabs 是 AI 语音生成和声音技术平台，提供语音生成、配音、API 等能力。

### 它服务谁

主要服务：

- 创作者
- 视频团队
- 播客团队
- 游戏/互动内容团队
- 企业本地化团队
- 开发者

### 它为什么值得学

它是典型的“底层能力 + 开发者生态 + 创作者工具”结合。

### 小团队可以学什么

1. API 能力可以被其他产品嵌入。
2. credit 计费适合成本随用量变化的 AI 产品。
3. 语音是视频、课程、播客、本地化的重要基础设施。

### 小团队不要学什么

不要做底层语音模型。

小团队更适合做上层工作流：

- 播客多语言版本
- 课程配音包
- 广告语音变体
- 有声书片段生成
- YouTube 本地化脚本 + 字幕 + 配音

### 迁移练习

请写：

```text
我可以基于哪些成熟 API 做垂直产品？
我的用户最终要的是声音文件、字幕包、视频，还是发布流程？
```

## 横向总结：10 个案例教我们的共同规律

| 规律 | 解释 | 对独立开发者的启发 |
| --- | --- | --- |
| 结果比功能重要 | 用户为最终结果付费，不为 AI 功能付费 | 文案里少说模型，多说结果 |
| 场景比通用重要 | 通用能力会被平台吞掉 | 选具体人群和具体任务 |
| 分发是护城河 | 产品好不等于有人看见 | 做 SEO、内容、社群或创始人品牌 |
| 信任影响转化 | 健康、法律、视频、文档都涉及信任 | 隐私、边界、样例、退款要清楚 |
| 工作流比单点强 | 单点工具容易被复制 | 把输入到输出串起来 |
| 可展示结果更容易传播 | 图片、视频、报告、对比图适合分享 | 设计输出时就考虑传播 |
| 小 B 比纯 C 更容易算账 | 小 B 更看 ROI | 优先找能帮用户赚钱/省钱的任务 |
| 买量不是万能 | 买量需要回本模型 | OPC 先找低成本渠道 |
| 合规不能补丁化 | 出海涉及隐私、订阅、AI 内容、版权 | 第一版就写清边界 |
| 小市场也值得做 | OPC 不需要一开始做独角兽 | 先追求可持续小闭环 |

## 学习者复盘模板

任选一个竞品，按下面模板写一页复盘。

```text
竞品名称：

1. 它服务谁？

2. 它替代了什么旧流程？

3. AI 在里面具体提效了哪一步？

4. 用户为什么愿意付费？

5. 它可能靠什么获客？

6. 它的信任/合规风险是什么？

7. 独立开发者可以学什么？

8. 独立开发者不该学什么？

9. 如果我要做一个更窄版本，我会服务谁？

10. 我的 MVP 可以只做哪一个动作？
```

## 课堂练习：把一个大赛道收窄

下面是几个大赛道，把它们收窄成适合 OPC 的方向。

### 泛图片生成

错误方向：

```text
做一个 AI 图片生成 App。
```

## 最后一页：案例学习的真正目的

拆竞品不是为了复制。

如果你只复制功能，你永远慢一步。  
如果你理解它为什么成立，你就能迁移到自己的细分场景。

真正要学的是：

```text
用户是谁
痛点是什么
旧流程是什么
AI 提效在哪里
结果怎么展示
信任怎么建立
渠道怎么获得
小团队怎么避开正面竞争
```

当你能用这 8 个问题看任何 AI 产品时，你就不再只是看热闹，而是在训练自己的产品判断力。
